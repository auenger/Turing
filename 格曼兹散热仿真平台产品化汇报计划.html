建立独立的任务建单流程

建模文件只是任务输入的一部分。业务信息、验收指标、调优边界和文件校验需要形成正式、可版本化的任务快照。

基本信息

填写客户、项目、产品型号、工况名称、负责人、运行环境和外部工单号，支持保存草稿。

验收要求

定义最高温度、关键监测点、风量、压降、收敛标准和允许回退范围。所有数值明确单位和来源。

调优边界

限定风扇、材料、TIM、散热片、风道和功耗等动作的可用范围，并定义哪些动作必须人工确认。

建模文件

独立完成大文件上传、哈希校验、模型结构预检和版本管理。原始文件始终保留，调整只作用于工作副本。

确认提交

汇总表单、模型文件、验收规则和调优边界。确认后冻结需求快照并进入求解队列。

将全自动调优改为人机协同

未达标时不再直接执行 Agent 生成的动作。系统先形成候选方案和待确认节点，工程师确认后再继续下一轮模型任务。

提交任务冻结表单与模型快照
基线求解调用真实 Icepak 或 Mock 环境
确定性判断按验收规则输出 PASS、FAIL 或异常
Agent 生成方案结合知识库和历史效果形成 2 至 4 个候选方案
人工确认节点Web 与钉钉同步待办。工程师对比证据、风险和预期影响，选择单一方案或全部驳回。
应用选中方案只修改工作副本，生成新模型版本
下一轮求解重新判断并保留完整迭代记录
系统执行 Agent 能力 人工决策

Agent 可以做什么

  • 分析不达标指标和可能原因
  • 引用内置知识与历史调优效果
  • 生成结构化候选方案并排序
  • 说明动作、风险、证据和预期影响

Agent 不能直接做什么

  • 未经确认直接修改工程模型
  • 自行降低或放宽验收指标
  • 绕过动作范围、预算和安全策略
  • 将 LLM 置信度作为工程安全结论

后续功能实施计划

实施顺序以业务依赖为依据，先建立任务和状态基础，再完成人工确认闭环，最后接入企业协同与产品化保障。

  • 工作台、任务列表和应用导航

  • 建模文件上传、校验和版本管理
  • 任务详情、轮次对比和报告中心
  • 执行状态、热判定和审批状态分离
  • 角色权限和租户数据边界

Agent 方案能力

  • 结构化诊断和候选方案合同
  • 知识引用、证据和风险说明
  • AUTO、CONFIRM、ADVISE_ONLY 策略校验
  • 历史效果回流和方案质量评估

人工确认闭环

  • 待办中心和方案对比界面
  • 选择、驳回、意见和超时处理
  • Temporal 持久等待与信号恢复
  • 审批幂等、模型版本和决策审计

钉钉 Agent Gateway

  • 钉钉身份与平台用户映射
  • 群聊、单聊与任务显式绑定
  • 待办卡片、方案确认和状态同步
  • 通知重试、签名校验、防重放和限流

产品化保障

  • 通知 Outbox 与失败告警
  • 业务操作和 Agent 行为审计
  • 异常恢复、重复操作和权限测试
  • 部署、监控、备份与回退机制

需要客户共同确认的事项

以下内容会直接决定页面表单、Agent 行为边界、审批流程和钉钉接入方式，建议在产品原型确认前形成明确结论。

01任务与验收标准

确认必填字段、单位、默认值、关键监测点、温度上限、风量、压降和正式收敛定义。

02Agent 动作权限

明确哪些动作允许自动执行，哪些必须人工确认，哪些只能展示为工程建议，以及对应参数范围。

03人工决策规则

确认候选方案数量、审批人范围、单选或会签、全部驳回后的处理方式和审批超时规则。

04异常处理方式

确认 INVALID、DIVERGED 和求解失败是否允许有限自动修复，还是直接生成工程异常待办。

05钉钉应用形态

确认内部企业应用或第三方应用、单聊和群聊范围、测试组织、机器人权限及互动卡片权限。

06数据与审计要求

确认文件和报告保留期、敏感数据展示范围，以及报告是否记录全部候选方案和人工决策原因。

最终形成可解释、可确认、可追溯的散热优化协作平台

保留现有 Icepak、Mock、Temporal、MinIO、PostgreSQL 和报告链路，重点补齐正式产品界面、人工决策节点、企业通知和治理能力。
